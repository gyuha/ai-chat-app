## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

**예외:**
- 사이드바 너비: 280px (고정)
- 터치 타겟 (모바일): 최소 44px (iOS HIG 준수)

**사용처 (Phase 1):**
- `xs(4px)`: 아이콘과 텍스트 간격
- `sm(8px)`: 대화 목록 아이템 내부 패딩, 버튼 내부 패딩
- `md(16px)`: 섹션 간 간격, 입력 필드 패딩
- `lg(24px)`: 사이드바 섹션 간 간격
- `xl(32px)`: 페이지 상하 패딩

| Dominant (60%) | `hsl(0 0% 3.9%)` — 다크 배경 | 메인 채팅 영역 배경 |

- 입력 필드 포커스 상태
- 활성 대화 아이템 배경
- 주요 CTA 버튼

## Phase 1 Interaction Specifications

### 대화 생성

1. 사용자가 "새 대화" 버튼 클릭
2. `useCreateConversation` mutation 호출 → IndexedDB에 새 대화 저장
3. TanStack Query 캐시 무효화 (`invalidateQueries(['conversations'])`)
4. TanStack Router로 `/chat/$conversationId` 네비게이션
5. 사이드바에서 새 대화가 최상단에 표시됨

### 대화 삭제

1. 사용자가 대화 아이템의 "삭제" 아이콘 클릭
2. `DeleteConversationDialog` 표시 (Radix UI Dialog)
3. 사용자 확인 시:
   - `useDeleteConversation` mutation 호출 → IndexedDB에서 삭제
   - TanStack Query 캐시 무효화
   - 현재 대화 페이지인 경우 홈(`/`)으로 리다이렉트
4. 사용자 취소 시 다이얼로그 닫기

### API 키 관리

1. **등록:**
   - API 키 입력 필드에 값 입력
   - "저장" 버튼 클릭 → IndexedDB settings 테이블에 저장
   - 성공 시 토스트 알림 "API 키가 저장되었습니다."

2. **변경:**
   - 기존 API 표시 (마스킹: `sk-••••••••`)
   - "변경" 클릭 → Input에 기존 값 표시
   - 수정 후 "저장" → IndexedDB 업데이트

3. **삭제:**
   - "삭제" 버튼 클릭 → 확인 다이얼로그
   - 확인 시 IndexedDB에서 제거
   - API 키 미등록 상태로 전환

### 기본 모델 선택

1. 드롭다운에서 모델 선택 (Radix UI Select)
2. "저장" 버튼 클릭 → IndexedDB settings 테이블에 저장
3. 이후 새 대화 생성 시 선택한 모델이 기본값으로 적용

### 시스템 프롬프트 설정

1. Textarea에 프롬프트 입력 (다중 행)
2. "저장" 버튼 클릭 → IndexedDB conversations 테이블의 `systemPrompt` 필드에 저장
3. 대화별 설정 vs 글로벌 설정:
   - **글로벌:** 설정 페이지에서 설정한 기본값
   - **대화별:** 특정 대화에서만 다른 프롬프트 사용 (Phase 3 이후 구현)
