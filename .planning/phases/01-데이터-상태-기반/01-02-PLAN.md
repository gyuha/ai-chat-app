`src/hooks/use-conversations.ts`에 다음 3개 훅을 구현:

1. `useConversations()` - TanStack Query `useQuery` 사용:
   - queryKey: `['conversations']`
   - queryFn: `db.conversations.orderBy('updatedAt').reverse().toArray()`
   - 반환 타입: `UseQueryResult<Conversation[]>`

2. `useCreateConversation()` - TanStack Query `useMutation` 사용:
   - mutationFn: `crypto.randomUUID()`로 id 생성, title은 '새 대화', modelId는 파라미터로 전달 (기본값: 'default'), createdAt/updatedAt은 `new Date()`로 설정
   - onSuccess: `queryClient.invalidateQueries({ queryKey: ['conversations'] })`
   - mutation 성공 후 TanStack Router `useNavigate`로 `/chat/${conversation.id}` 이동

3. `useDeleteConversation()` - TanStack Query `useMutation` 사용:
   - mutationFn: 반드시 메시지 먼저 삭제 후 대화 삭제 (cascade):
     ```typescript
     await db.messages.where('conversationId').equals(id).delete();
     await db.conversations.delete(id);
     ```
   - onSuccess: conversations 쿼리 무효화
   - 현재 대화 페이지인 경우 TanStack Router `useNavigate`로 `/` 리다이렉트

TanStack Query의 `useQueryClient`를 사용하여 캐시 무효화. `useNavigate`는 훅 내부가 아닌 컴포넌트에서 호출해야 하므로, mutation의 onSuccess에서 navigate를 직접 호출하지 않고 컴포넌트 측에서 `onSuccess` 콜백으로 처리하도록 설계. 훅은 mutation 객체만 반환.

주의: Dexie v4에서 `EntityTable<Conversation, 'id'>` 사용, `Table` 사용 금지.
