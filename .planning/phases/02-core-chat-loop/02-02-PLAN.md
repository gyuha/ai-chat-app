OpenRouter uses OpenAI-compatible SSE format:
```
data: {"choices":[{"delta":{"content":"Hello"}}]}

```

<action>
Create src/lib/openrouter.ts with streaming function:

1. Export interface for streaming options:
```typescript
export interface StreamOptions {
  apiKey: string;
  model: string;
  messages: Array<{ role: 'user' | 'assistant' | 'system'; content: string }>;
  signal: AbortSignal;
}
```

2. Export async function streamChat(options: StreamOptions):
   - Destructure apiKey, model, messages, signal from options
   - POST to `${OPENROUTER_API_BASE}/chat/completions`
   - Headers: Content-Type: application/json, Authorization: Bearer ${apiKey}
   - Body: { model, messages, stream: true }
   - Set signal on fetch request
   - If !response.ok, throw error with status
   - If !response.body, throw error 'ReadableStream not supported'

3. Stream parsing (based on RESEARCH.md Pattern 1):
   - Get reader from response.body.getReader()
   - Create TextDecoder
   - Let buffer = ''
   - While true:
     - read() from reader
     - If done, break
     - Decode value with { stream: true }
     - Append to buffer
     - Split buffer by '\n'
     - Keep last incomplete line in buffer
     - For each complete line:
       - If starts with 'data: ', extract data after 'data: '
       - If data === '[DONE]', return
       - Try parse JSON
       - Extract delta.content from parsed.choices[0].delta.content
       - If delta, yield delta
   - Finally: reader.releaseLock()

4. Add HTTP-Referer and X-Title headers per OpenRouter requirements
