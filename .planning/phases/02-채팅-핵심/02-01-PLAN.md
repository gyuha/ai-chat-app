      - headers: `{ "Content-Type": "application/json", "Authorization": "Bearer ${apiKey}" }`
      - body: `{ model: modelId, messages: apiMessages, stream: true }`

      - response.ok가 아니면 에러 throw (error.status에 HTTP 상태 코드 설정)
      - response.body.getReader()로 ReadableStream 읽기

      - "data: " 접두사 확인, "[DONE]"이면 onComplete 호출 후 return
      - JSON 파싱 후 `choices[0].delta.content`가 있으면 fullContent에 누적하고 onToken 콜백 호출

  <action>
    src/hooks/use-chat-stream.ts 생성:

    - `UseChatStreamReturn` 인터페이스 정의:
      ```typescript
      {
        streamingContent: string;
        isStreaming: boolean;
        error: Error & { status?: number } | null;
        sendMessage: (messages: Pick<Message, "role" | "content">[]) => Promise<string>;
        stopStreaming: () => void;
      }
      ```

    - `useChatStream(modelId: string, systemPrompt: string | null)` 훅 구현:
      - `useSetting(SETTINGS_KEYS.API_KEY)`로 API 키 조회
      - `useState<string>("")`로 streamingContent 관리
      - `useState<boolean>(false)`로 isStreaming 관리
      - `useState<Error & { status?: number } | null>(null)`로 error 관리
      - `useRef<AbortController | null>(null)`로 abortRef 관리

    - `sendMessage` 콜백 (useCallback):
      - API 키가 없으면 `throw new Error("API 키가 설정되지 않았습니다.")`
      - 이전 AbortController가 있으면 abort() 호출
      - 새 `AbortController` 생성하여 abortRef에 저장
      - isStreaming = true, error = null, streamingContent = "" 초기화
      - `streamChatCompletion(apiKey, modelId, messages, systemPrompt, controller.signal, callbacks)` 호출
      - callbacks.onToken: `setStreamingContent(prev => prev + token)`
      - callbacks.onComplete: fullContent 반환 (호출자에서 처리)
      - try/catch/finally:
        - catch: AbortError면 무시, 그 외는 setError 후 throw
        - finally: setIsStreaming(false)

    - `stopStreaming` 콜백 (useCallback):
      - `abortRef.current?.abort()`
      - `abortRef.current = null`
      - `setIsStreaming(false)`

    - 의존성: `[apiKey, modelId, systemPrompt]`

    - 반환: `{ streamingContent, isStreaming, error, sendMessage, stopStreaming }`

    **주의:**
    - sendMessage는 완성된 fullContent 문자열을 반환해야 함 (호출자가 IndexedDB에 저장)
    - AbortError는 error 상태에 설정하지 않음 (사용자 의도적 중단이므로)
    - error 타입에 status 속성을 포함하여 호출자가 HTTP 상태 코드별 분기 가능
