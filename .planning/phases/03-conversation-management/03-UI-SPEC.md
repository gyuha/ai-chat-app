### Focal Point

**사이드바 대화 목록**의 시각적 앵커는 다음과 같이 선언한다:

| 상태 | Focal Point | 위치 |
|------|-------------|------|

| 상태 | Focal Point | 위치 |
|------|-------------|------|
| API 키 미설정 | API 키 안내 카드 | 화면 중앙 |
| API 키 설정됨, 대화 없음 | 시작 안내 + 새 대화 CTA | 화면 중앙 |
| 대화가 선택됨 | 채팅 메시지 영역 | 화면 중앙 |

## Component Inventory

### Sidebar (CONV-01, CONV-02, CONV-03)
- **Component:** Left sidebar with conversation list
- **States:** Empty (no conversations), Has conversations, Conversation selected
- **Behavior:** "새 대화" button creates new conversation, list sorted newest first
- **Copy:** "새 대화" (button), "저장된 대화가 없습니다" (empty state)

### Conversation List Item (CONV-02, CONV-03, CONV-04)
- **Component:** Button/row in sidebar
- **States:** Default, Hover, Active (selected), Editing title
- **Behavior:** Click to select, double-click to edit title, show delete on hover (mobile: long press)
- **Active indicator:** bg-accent/10 with accent text color
- **Hover indicator:** bg-accent/5

### Title Edit Mode (CONV-04)
- **Component:** Inline input replacing conversation title
- **States:** View mode (showing title), Edit mode (input focused)
- **Behavior:** Click title to edit, Enter to save, Escape to cancel, auto-save on blur
- **Copy:** Input placeholder "제목 수정...", save on Enter

### Delete Confirmation Dialog (CONV-05)
- **Component:** AlertDialog (shadcn alert-dialog)
- **Trigger:** Hover delete button or swipe action
- **States:** Closed, Open (showing confirmation)
- **Copy:** "대화 삭제" (title), "정말로 이 대화를 삭제하시겠습니까? 이 작업은 되돌릴 수 없습니다." (description), "삭제" (confirm button, destructive), "취소" (cancel button)
- **Behavior:** Confirm triggers delete mutation, closes dialog

### Theme Toggle (UI-03)
- **Component:** DropdownMenu item or IconButton in sidebar header
- **States:** Light mode active, Dark mode active, System mode active
- **Icon:** Sun (light), Moon (dark), Monitor (system)
- **Copy:** "라이트 모드", "다크 모드", "시스템 설정"

### Mobile Sidebar Toggle (CONV-06)
- **Component:** Hamburger button in mobile header + backdrop overlay
- **States:** Sidebar closed, Sidebar open
- **Behavior:** Tap hamburger opens sidebar with backdrop, tap backdrop or X closes
- **Already implemented in:** chat-layout.tsx (hamburger + overlay)

### Home Empty State (UI-04)
- **Component:** Centered card with icon + heading + body + CTA
- **States:** API key not set, API key set but no conversation selected
- **When API key not set:** "OpenRouter API 키가 필요합니다" heading, "무료 AI 모델과 대화하려면 API 키를 입력해 주세요." body, "API 키 입력" CTA (navigates to /settings)
- **When API key set, no conversation:** "대화를 시작하세요" heading, "질문을 입력해 AI와 대화를 시작하세요." body, "새 대화" CTA (creates new conversation)
- **Layout:** Vertically centered, max-width 400px, 24px padding

### Home Chat Ready State (UI-04 continuation)
- **Component:** Show selected conversation or prompt suggestion cards
- **When conversation selected:** Show chat messages (existing behavior from Phase 1)
- **When no conversation selected:** Show empty state above
